# Screen Wireframes

Date: 2026-05-28

These are text wireframes for the B2B Corporate L&D MVP. They define screen intent and core controls before coding.

## 1. Dashboard

Purpose:

Show active training projects and operational status.

Content:

- project list
- project status
- pending SME reviews
- content readiness
- active cohorts
- open SME alerts
- report export shortcut

Primary actions:

- create project
- open project
- view alerts

## 2. Create Training Project

Purpose:

Capture minimum project setup.

Fields:

- program name
- business goal
- target department
- expected learner count
- target completion date
- known learner roles
- assigned SME

Primary actions:

- save draft
- continue to sources

## 3. Source Upload

Purpose:

Collect source material for requirement intelligence.

Content:

- upload area
- pasted text input
- source list
- extraction status
- active/excluded toggle
- extracted text preview

Primary actions:

- upload source
- paste source
- retry extraction
- generate program plan

## 4. Program Plan Review

Purpose:

Review AI-generated requirement summary, assumptions, conflicts, TOC, role tracks, and assessment blueprint.

Layout:

- left: plan sections
- center: selected section details
- right: comments and source references

Sections:

- requirement summary
- assumptions
- conflicts
- learner roles
- TOC
- prerequisite map
- assessment blueprint

Primary actions:

- add comment
- request changes
- approve TOC
- lock TOC

## 5. Content Factory

Purpose:

Generate and review training artifacts from locked TOC.

Content:

- locked TOC summary
- artifact list
- artifact status
- preview panel
- SME comments

Artifacts:

- slide outline
- facilitator notes
- learner handout
- exercises
- quiz bank
- practical task
- rubric

Primary actions:

- generate artifact
- submit for review
- approve artifact
- request changes
- export package

## 6. Cohort Setup

Purpose:

Prepare lightweight learner delivery.

Fields:

- cohort name
- start date
- end date
- instructor
- learner list
- assigned role track

Primary actions:

- save cohort
- enroll learners
- publish approved content

## 7. Learner Module View

Purpose:

Allow learner to consume assigned approved content and complete assessment.

Content:

- module list
- current module content
- quiz
- practical task submission
- help request button
- progress indicator

Primary actions:

- start module
- complete module
- submit quiz
- submit practical task
- request help

## 8. Cohort Progress

Purpose:

Give L&D admin, instructor, and SME operational visibility.

Content:

- learner list
- role track
- module progress
- quiz score
- practical task status
- alerts
- risk indicator

Primary actions:

- open learner detail
- create SME alert
- resolve alert
- export progress report

## 9. SME Alert Detail

Purpose:

Help SME/instructor intervene quickly.

Content:

- learner
- cohort
- module/topic
- trigger reason
- evidence
- previous attempts
- recommended action
- notes

Primary actions:

- acknowledge
- add note
- assign remediation
- resolve
- dismiss

## 10. Reports

Purpose:

Export leadership-ready B2B reports.

Reports:

- program readiness report
- learner progress report
- SME intervention log
- assessment summary
- time-saved/ROI report

Primary actions:

- preview report
- export PDF
- export CSV
